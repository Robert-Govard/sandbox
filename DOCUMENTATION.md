# Документация проекта Sandbox

## Обзор проекта

**Sandbox** — это мультиплеерная песочница, созданная компанией Facepunch с использованием платформы s&box. Проект предоставляет игрокам возможность творить, строить и взаимодействовать в трехмерном пространстве с набором инструментов и оружия.

- **Версия**: 1.0
- **Разработчик**: Facepunch
- **Лицензия**: MIT License
- **Платформа**: s&box
- **Язык программирования**: C# (.NET 10.0)
- **Репозиторий**: https://github.com/Facepunch/sandbox

## Основные системы

### 1. Система игрока (Player/)

**Основной класс**: `Player.cs`

Класс `Player` управляет всеми аспектами игрока:

- **Здоровье и броня**: Управление HP и Armour с синхронизацией по сети
- **Камера**: Обработка камеры от первого лица
- **Инвентарь**: Система хранения и переключения оружия
- **Урон**: Обработка получения урона и смерти
- **Движение**: Интеграция с PlayerController
- **События**: Система событий игрока (прыжок, приземление, смерть)

**Ключевые компоненты**:
- `PlayerController` — контроллер движения
- `PlayerInventory` — инвентарь оружия
- `PlayerData` — данные игрока (статистика, киллы)
- `PlayerLoadout` — начальная экипировка
- `PlayerObserver` — режим наблюдателя после смерти

**Основные методы**:
- `OnDamage(in DamageInfo dmg)` — обработка урона
- `Kill(in DamageInfo d)` — обработка смерти
- `EquipBestWeapon()` — экипировка лучшего оружия
- `SwitchWeapon<T>()` — переключение оружия

### 2. Система оружия (Weapons/)

**Базовые классы**:
- `ScreenWeapon` — оружие с экраном прицеливания
- `CameraWeapon` — оружие с камерой
- `BaseCarryable` — базовый класс для переносимых объектов

**Типы оружия**:
- **Пистолеты**: Glock, Colt 1911
- **Автоматы**: M4A1, MP5
- **Дробовики**: Shotgun
- **Снайперские**: Sniper
- **Тяжелое**: RPG, Hand Grenade
- **Инструменты**: PhysGun, ToolGun, Crowbar

### 3. ToolGun (Инструментальный пистолет)

**Основной класс**: `Toolgun.cs`

ToolGun — это многофункциональный инструмент с различными режимами:

**Режимы инструментов** (Code/Weapons/ToolGun/Modes/):
- **Weld** — сварка объектов
- **Rope** — создание веревок
- **Elastic** — создание пружин
- **Slider** — создание ползунков
- **BallSocket** — шарнирные соединения
- **Hydraulic** — гидравлические системы
- **Thruster** — двигатели
- **Hoverball** — левитирующие шары
- **KeepUpright** — сохранение ориентации
- **Balloon** — воздушные шары
- **Wheel** — колеса
- **Remover** — удаление объектов
- **NoCollide** — отключение коллизий
- **Unbreakable** — делает объект неразрушимым
- **Mass** — изменение массы
- **Resizer** — изменение размера
- **Trail** — создание следов
- **Decal** — декали
- **Emitter** — эмиттеры частиц
- **Duplicator** — дубликатор объектов
- **Stacker** — стекание объектов
- **Linker** — связывание объектов

**Основной функционал**:
- `SetToolMode(string name)` — переключение режима
- `GetCurrentMode()` — получение текущего режима
- `CreateToolComponents()` — создание компонентов инструментов
- Сетка привязки (SnapGrid) для точного позиционирования

### 4. Система спавна (Spawner/)

**Интерфейсы**:

### 5. Система сохранения (Save/)

**Основные методы**:

## Архитектурные паттерны

### 1. Компонентная система

Проект использует компонентную архитектуру, где игровые объекты строятся из набора компонентов:
- `Component` — базовый класс для всех компонентов
- `GameObject` — контейнер для компонентов
- `[RequireComponent]` — атрибут для обязательных компонентов

### 2. Синхронизация по сети

- `[Sync]` — атрибут для синхронизации свойств
- `[Rpc]` — атрибут для удаленных вызовов процедур
- `Networking.IsHost` — проверка хоста
- `Network.Owner` — владелец сети

### 3. Система событий

- `IPlayerEvents` — события игрока
- `ISaveEvents` — события сохранения
- `Local.IPlayerEvents.PostToGameObject()` — локальные события
- `Global.IPlayerEvents.Post()` — глобальные события

### 4. Атрибуты свойств

- `[Property]` — редактируемые свойства
- `[Range]` — ограничение диапазона
- `[Sync]` — синхронизация по сети

## Система отладки хитбоксов (Hitbox Debug System)

В проекте реализована система отладки для визуализации хитбоксов игроков и NPC. Все команды помечены флагом `ConVarFlags.Cheat` и требуют включенного режима читерства.

### Консольные команды

- `hitbox` — включить/выключить все хитбоксы
- `hitbox_players` — включить/выключить хитбоксы игроков (зеленые)
- `hitbox_npcs` — включить/выключить хитбоксы NPC (красные)
- `hitbox_help` — показать справку по командам

### Примеры использования

```
# Включить отображение всех хитбоксов
hitbox

# Включить только хитбоксы игроков
hitbox_players

# Включить только хитбоксы NPC
hitbox_npcs

# Показать помощь
hitbox_help
```

### Техническая реализация

Файл: `Code/Utility/HitboxDebugSystem.cs`

Система использует:
- `Component` с автоматическим созданием через `GetOrCreate()`
- `DebugOverlay.Box()` для рендеринга отладочных прямоугольников
- `Scene.GetAllComponents<Player>()` и `Scene.GetAllComponents<Npc>()` для поиска объектов
- Оптимизированный рендеринг только при включенных флагах

## Target Dummy NPC

Специальный NPC-мишень для тестирования хитбоксов. Стоит на месте, получает урон и автоматически респавнится после гибели.

### Характеристики

- Здоровье: 200 (настраиваемое, 1-500)
- Время респавна: 5 секунд (настраиваемое)
- Поведение: стоит на месте, поворачивает голову к ближайшему видимому объекту
- При гибели: создаёт рэгдолл и респавнится на месте спавна

### Способ спавна

**Через консоль:**
```
spawn_target_dummy
```

**Через стандартную команду спавна:**
```
spawn sent:entities/sents/npc/target_dummy
```

**Через спавн-меню:** категория "Npc" → "Target Dummy"

### Совместимость с хитбокс-системой

Target Dummy наследуется от базового класса `Npc`, поэтому автоматически отображается при включении команды `hitbox_npcs`. NPC имеет компонент `ModelHitboxes` для точных хитбоксов модели citizen.

### Файлы

- Код: `Code/Npcs/TargetDummy/TargetDummyNpc.cs`
- Префаб: `Assets/entities/sents/npc/target_dummy.prefab`
- Спавн-меню: `Assets/entities/sents/npc/target_dummy.sent`
